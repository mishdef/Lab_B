# Проєкт (частина В)

## Тема

УСПАДКУВАННЯ. АБСТРАКТНІ КЛАСИ. ПОЛІМОРФІЗМ

## Опис проєкту

Предметна область: Система управління проєктами компанії.

Цей проєкт є реалізацією (частина В) архітектури, спроєктованої у частині А. На цьому етапі каркас класів був наповнений логікою, а також були реалізовані ключові принципи ООП: успадкування, абстрактні класи та поліморфізм.

## Ключові особливості (частина В)

У цій версії проєкту було реалізовано наступні концепції та механізми:

**Абстрактний клас та успадкування**:
*   Створено абстрактний клас `User`, який узагальнює сутність користувача та містить спільні властивості (`Name`, `Email`) та абстрактний метод `GetInfo()`.
*   Класи `CEO`, `Employee` та `ProjectManager` тепер успадковуються від `User` та надають власну реалізацію методу `GetInfo()`.

**Поліморфізм**:
*   Поліморфізм реалізовано через перевизначення методу `GetInfo()` у похідних класах. Це дозволяє уніфіковано працювати з об'єктами різних типів користувачів через базовий тип `User`.
 *   Продовжується використання інтерфейсів `IUserInfo` та `IPrintable` для роботи з об'єктами незалежно від їх конкретної реалізації.

**Реалізація інтерфейсів .NET**:
*   Клас `Task` реалізує інтерфейс `ICloneable`, що дозволяє створювати копії завдань.
*   Клас `ProjectBoard` реалізує `IEnumerable<Task>`, що дозволяє легко ітерувати список його завдань за допомогою циклу `foreach`.

**Реалізація бізнес-логіки**:
*   Всі "заглушки" з частини А замінено на повноцінну логіку.
*   Клас `PermissionService` тепер реалізує перевірку прав доступу на основі ролі користувача (`CEO`, `ProjectManager` тощо).
*   Реалізовано систему користувацьких винятків (`InfoException`, `WarningException`, `ErrorException`) для кращої обробки помилок та інформування користувача.

## Основні функції програми

Програма надає повноцінний консольний інтерфейс для управління проєктами компанії:

*   **Управління компанією**: створення компанії, додавання/видалення співробітників, створення/видалення проєктних дошок (доступно лише для `CEO`).
*   **Управління проєктами**: перегляд списку проєктів, додавання/видалення завдань у проєкті (доступно для `CEO` та `ProjectManager`).
*   **Управління завданнями**: призначення завдань на співробітників, зміна їх статусу, клонування існуючих завдань.
*   **Система ролей та доступу**: програма чітко розмежовує можливості користувачів залежно від їхньої ролі. Спроби виконати дію без відповідних прав коректно обробляються.
*   **Зміна користувача**: можливість "перелогінитися" під іншим співробітником для тестування системи ролей.
